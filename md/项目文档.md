# 图书管理系统 - 技术文档

## 1. 项目概述

### 1.1 项目简介

图书管理系统（VueBookManagerSystem）是一个基于 Vue3 + Element-Plus 构建的纯前端单页应用（SPA），提供完整的图书借阅管理功能，包含读者端和管理端两个独立界面。

### 1.2 核心功能

| 模块 | 读者端功能 | 管理端功能 |
|------|----------|----------|
| 认证 | 用户登录、注册、验证码 | 管理员登录、记住密码 |
| 图书 | 图书查询、借阅 | 图书管理（增删改查）、批量采购 |
| 借阅 | 借阅记录、在线还书 | 借阅历史管理、状态追踪 |
| 用户 | 个人借阅统计 | 用户管理（增删改查） |
| 系统 | 反馈提交 | 系统配置、操作日志 |

### 1.3 技术特点

- **纯前端架构**：基于 localStorage 实现数据持久化，无需后端服务器
- **角色权限分离**：读者端与管理端独立路由，支持角色级访问控制
- **响应式设计**：适配桌面端与移动端，支持侧边栏折叠
- **现代化UI**：基于 Element-Plus，参考 RuoYi 风格设计系统

## 4. 核心模块详解

### 4.1 路由管理 (`src/router/index.js`)

#### 4.1.1 路由结构

```javascript
// 读者端路由
/                      → 重定向到 /login
/login                 → 读者登录页（guest）
/book-query            → 图书查询（需要用户登录）
/borrow-info           → 借阅信息（需要用户登录）

// 管理端路由
/admin/login           → 管理员登录页（guest）
/admin                 → 管理端布局（需要管理员登录）
  ├── user-manage      → 用户管理
  ├── borrow-history   → 借阅历史
  ├── book-manage      → 图书管理
  ├── purchase-manage  → 采购管理
  └── system-manage    → 系统管理

// 公共路由
/:pathMatch(.*)*      → 404页面
```

#### 4.1.2 路由守卫逻辑

路由守卫实现了以下访问控制策略：

| 场景 | 处理逻辑 |
|------|---------|
| 已登录用户访问登录页 | 自动跳转到对应首页（管理员→管理端，读者→图书查询） |
| 未登录用户访问受保护页面 | 跳转到对应登录页（管理端页面→管理员登录，读者页面→读者登录） |
| 角色权限不匹配 | 跳转到当前角色的首页（读者访问管理端→图书查询，管理员访问读者端→用户管理） |

#### 4.1.3 路由元信息

```javascript
meta: {
  title: '页面标题',           // 页面标题
  requiresAuth: true,         // 是否需要登录
  role: 'admin' | 'user',     // 允许访问的角色
  guest: true                 // 是否为游客页面（登录页）
}
```

### 4.2 状态管理 (`src/stores/`)

#### 4.2.1 authStore - 认证状态

**State:**
- `currentUser`: 当前登录用户对象
- `rememberedAdmin`: 记住的管理员账号信息

**Getters:**
- `isLoggedIn`: 是否已登录
- `isAdmin`: 是否为管理员
- `isUser`: 是否为普通用户

**Actions:**
- `setCurrentUser(user)`: 设置当前用户
- `logout()`: 登出（清除用户信息）
- `rememberAdmin(username, password)`: 记住管理员密码
- `clearRememberedAdmin()`: 清除记住的密码

#### 4.2.2 bookStore - 图书状态

**State:**
- `books`: 图书列表
- `filters`: 筛选条件（keyword, category, author, publisher, priceRange）

**Getters:**
- `filteredBooks`: 筛选后的图书列表
- `availableBooks`: 可借阅图书列表
- `hotBooks`: 热门图书（借阅次数前5）

**Actions:**
- `fetchBooks()`: 从localStorage加载图书
- `addBook(bookData)`: 新增图书
- `updateBook(id, bookData)`: 更新图书信息
- `deleteBook(id)`: 删除图书
- `incrementBorrowCount(id)`: 增加借阅次数
- `decrementBorrowCount(id)`: 减少借阅次数

#### 4.2.3 userStore - 用户状态

**State:**
- `users`: 用户列表
- `filters`: 筛选条件（keyword, phone, address）

**Getters:**
- `filteredUsers`: 筛选后的用户列表
- `bestReaders`: 优秀读者（借阅次数前5）

**Actions:**
- `fetchUsers()`: 从localStorage加载用户
- `addUser(userData)`: 新增用户
- `updateUser(id, userData)`: 更新用户信息
- `deleteUser(id)`: 删除用户
- `login(username, password, role)`: 用户登录验证
- `register(userInfo)`: 用户注册

#### 4.2.4 borrowStore - 借阅状态

**State:**
- `borrows`: 借阅记录列表
- `filters`: 筛选条件（bookName, category, author, status）

**Getters:**
- `filteredBorrows`: 筛选后的借阅记录
- `currentUserBorrows`: 当前用户的借阅记录
- `overdueBorrows`: 过期借阅记录

**Actions:**
- `fetchBorrows()`: 从localStorage加载借阅记录
- `addBorrow(book, user, borrowDays)`: 新增借阅记录
- `returnBook(borrowId)`: 归还图书
- `updateBorrow(id, data)`: 更新借阅记录

#### 4.2.5 purchaseStore - 采购状态

**State:**
- `purchases`: 采购记录列表
- `filters`: 筛选条件（bookName, amountRange）

**Getters:**
- `filteredPurchases`: 筛选后的采购记录

**Actions:**
- `fetchPurchases()`: 从localStorage加载采购记录
- `addPurchase(purchaseData)`: 新增采购单
- `updatePurchase(id, purchaseData)`: 更新采购单
- `deletePurchase(id)`: 删除采购单
- `updateStatus(id, status)`: 更新采购状态

#### 4.2.6 configStore - 系统配置

**State:**
- `config`: 系统配置对象

**配置项:**
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| borrowDays | 30 | 默认借阅天数 |
| pageSize | 10 | 默认分页大小 |
| enableCaptcha | true | 是否启用验证码 |
| adminPassword | YWRtaW4xMjM= | 管理员密码（Base64编码） |

**Actions:**
- `fetchConfig()`: 加载配置
- `updateConfig(newConfig)`: 更新配置
- `resetConfig()`: 重置为默认配置

#### 4.2.7 logStore - 操作日志

**功能:** 记录所有关键操作，便于审计追踪

**日志记录格式:**
```javascript
{
  id: '唯一标识',
  operator: '操作人',
  actionType: '操作类型(create/update/delete/borrow/return)',
  targetType: '操作对象(book/user/borrow/purchase)',
  description: '操作描述',
  createdAt: '操作时间'
}
```

**全局日志记录函数:**
```javascript
addLog(actionType, targetType, description)  // 在任意store中调用
```

### 4.3 工具模块 (`src/utils/`)

#### 4.3.1 storage.js - localStorage封装

**存储Key常量:**
```javascript
STORAGE_KEYS = {
  BOOKS: 'bms_books',
  USERS: 'bms_users',
  BORROWS: 'bms_borrows',
  PURCHASES: 'bms_purchases',

  REMEMBERED_ADMIN: 'bms_rememberedAdmin',
  FEEDBACKS: 'bms_feedbacks'

```

**密码编解码:**
- `encodePassword(password)`: Base64编码（支持Unicode字符）
- `decodePassword(encoded)`: Base64解码

#### 4.3.2 init.js - 数据初始化

系统首次加载时自动执行，将Mock数据写入localStorage：
```javascript
initData()  // 检查localStorage是否有数据，无则初始化
```

#### 4.3.3 id.js - ID生成工具

- `generateId()`: 生成唯一ID（优先使用crypto.randomUUID）
- `generateBookNo(index)`: 生成图书编号（格式：BK-20260001）
- `formatDate(date)`: 格式化日期为 YYYY-MM-DD
- `formatDateTime(date)`: 格式化日期时间为 YYYY-MM-DD HH:mm:ss

#### 4.3.4 validate.js - 表单验证

**预定义验证规则:**
- `required(message)`: 必填验证
- `phoneRule`: 手机号格式验证（1开头，11位）
- `isbnRule`: ISBN格式验证（10位或13位数字）
- `positiveIntegerRule`: 正整数验证
- `priceRule`: 价格验证（正数）

**验证函数:**
- `isValidPhone(value)`: 验证手机号
- `isValidISBN(value)`: 验证ISBN
- `isEmpty(value)`: 验证是否为空
- `isValidNumber(value)`: 验证是否为有效数字
- `isPositiveInteger(value)`: 验证是否为正整数

## 5. 页面功能说明

### 5.1 登录页面

#### 5.1.1 读者登录 (`Login.vue`)

**功能:**
- 账号密码登录
- 验证码验证（可配置开关）
- 用户注册（用户名、密码、手机号）
- 管理员登录入口

**交互流程:**
1. 用户输入账号密码（可选验证码）
2. 表单验证通过后调用 `userStore.login()`
3. 验证成功后调用 `authStore.setCurrentUser()`
4. 跳转至 `/book-query`

**验证码机制:**
- 随机生成4位数字验证码
- 点击可刷新验证码
- 登录失败自动刷新

#### 5.1.2 管理员登录 (`AdminLogin.vue`)

**功能:**
- 管理员账号密码登录
- 记住密码功能
- 返回读者登录入口

**登录验证逻辑:**
1. 优先通过 `userStore.login()` 验证
2. 若用户表中无此管理员，回退到系统默认管理员账号
3. 验证成功后设置登录状态并跳转

### 5.2 读者端页面

#### 5.2.1 图书查询 (`BookQuery.vue`)

**功能:**
- 多条件搜索（关键词、类型、出版社）
- 图书列表展示（图书号、类型、名称、作者、出版社、数量）
- 借阅操作
- 分页支持
- 侧边栏（热门图书、优秀读者、反馈提交）

**借阅流程:**
1. 用户点击"借阅"按钮
2. 检查图书可借数量
3. 调用 `bookStore.incrementBorrowCount()` 更新图书状态
4. 调用 `borrowStore.addBorrow()` 创建借阅记录
5. 调用 `userStore.updateUser()` 更新用户借阅次数

#### 5.2.2 借阅信息 (`BorrowInfo.vue`)

**功能:**
- 当前用户借阅记录列表
- 状态筛选（全部、借阅中、已过期）
- 在线还书
- 过期记录高亮显示

**还书流程:**
1. 用户点击"还书"按钮
2. 弹出确认对话框
3. 调用 `borrowStore.returnBook()` 更新借阅状态
4. 调用 `bookStore.decrementBorrowCount()` 更新图书库存
5. 更新用户借阅次数

**过期判断:**
- status === 'overdue'
- 或 status === 'borrowing' 且 dueDate < 当前时间

## 6. 数据模型

### 6.1 图书模型 (`Book`)

```javascript
{
  id: 'string',           // 唯一标识
  bookNo: 'string',       // 图书编号（格式：BK-20260001）
  name: 'string',         // 图书名称
  category: 'string',     // 分类（科技/文学/科幻/历史/教育）
  author: 'string',       // 作者
  publisher: 'string',    // 出版社
  isbn: 'string',         // ISBN编号
  price: 'number',        // 价格
  totalCount: 'number',   // 总数量
  borrowCount: 'number',  // 已借数量
  status: 'string',       // 状态（available/unavailable）
  description: 'string',  // 描述
  createdAt: 'string'     // 创建时间（ISO格式）
}
```

### 6.2 用户模型 (`User`)

```javascript
{
  id: 'string',           // 唯一标识
  username: 'string',     // 用户名
  password: 'string',     // 密码（Base64编码）
  phone: 'string',        // 手机号
  age: 'number',          // 年龄
  address: 'string',      // 地址
  role: 'string',         // 角色（admin/user）
  borrowCount: 'number',  // 借阅次数

  registeredAt: 'string'  // 注册时间（ISO格式）
}
```

### 6.3 借阅模型 (`Borrow`)

```javascript
{
  id: 'string',           // 唯一标识
  bookId: 'string',       // 图书ID
  bookNo: 'string',       // 图书编号
  bookName: 'string',     // 图书名称
  bookCategory: 'string', // 图书分类
  bookAuthor: 'string',   // 图书作者
  bookPrice: 'number',    // 图书价格
  userId: 'string',       // 用户ID
  userAccount: 'string',  // 用户账号
  userName: 'string',     // 用户姓名
  borrowDate: 'string',   // 借阅日期（ISO格式）
  dueDate: 'string',      // 截止日期（ISO格式）
  returnDate: 'string',   // 归还日期（ISO格式，未归还为null）
  status: 'string'        // 状态（borrowing/overdue/returned）
}
```

### 6.4 采购模型 (`Purchase`)

```javascript
{
  id: 'string',           // 唯一标识

  amount: 'number',       // 总金额

  responsible: 'string',  // 负责人
  status: 'string',       // 状态（pending/approved/completed）
```

### 6.5 反馈模型 (`Feedback`)

## 8. 开发指南

### 8.1 新增页面流程

1. **创建视图组件**: 在 `src/views/` 下创建 `.vue` 文件
2. **配置路由**: 在 `src/router/index.js` 中添加路由配置
3. **创建状态管理**（如需要）: 在 `src/stores/` 下创建 `xxx.js`
4. **添加导航菜单**（管理端）: 在 `src/views/Layout.vue` 的 `el-menu` 中添加菜单项

### 8.5 注意事项

1. **localStorage限制**: 数据存储在浏览器本地，清除浏览器数据会丢失所有信息
2. **密码安全性**: 当前使用Base64编码，生产环境建议使用后端API + HTTPS
3. **数据一致性**: 修改数据后需同步更新相关联的Store（如借阅时同时更新图书和用户）
4. **路由守卫**: 所有受保护页面需配置 `requiresAuth` 和 `role` 元信息
