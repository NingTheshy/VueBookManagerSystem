## 2. 项目背景与概述

### 2.1 项目背景

随着信息化技术的普及，传统图书馆的人工管理模式已无法满足高效、便捷的图书管理需求。本项目旨在开发一个基于 **纯前端技术栈** 的图书管理系统，为图书馆管理员和读者提供图书管理、借阅追踪、采购管理、用户信息维护及系统管理等一站式功能。

### 2.2 项目目标

- 构建一套完整、可交互的图书管理系统前端界面。
- 实现图书的增删改查、借阅归还、采购订单、用户管理等核心业务流程的页面化操作。
- 遵循前端开发最佳实践，采用组件化、模块化设计思想。
- 满足课程设计评分标准中的所有技术考核点。

### 2.3 项目范围

本项目 **仅包含前端部分**，不涉及任何后端服务或数据库开发。所有数据均以 **浏览器本地存储（localStorage / IndexedDB）** 或 **Mock 数据** 的方式进行模拟，确保系统的纯前端性质。

### 2.4 关键术语定义

| 术语         | 说明                                                         |
| :----------- | :----------------------------------------------------------- |
| 图书管理     | 对图书基本信息进行新增、编辑、删除、条件检索的功能模块       |
| 借阅历史     | 记录图书借出和归还操作、展示借阅明细的功能模块               |
| 采购管理     | 对图书馆采购需求、采购订单进行记录和跟踪的功能模块           |
| 用户信息     | 对读者用户进行查询检索、新增、编辑、删除的功能模块           |
| 系统管理     | 系统设置、权限管理、日志记录的综合管理模块                   |
| 纯前端       | 不依赖任何后端服务，所有数据处理在浏览器端完成               |
| Mock 数据    | 预置在前端代码中的模拟数据，用于模拟后端接口返回             |
| localStorage | 浏览器提供的键值对本地持久化存储机制                         |

## 3. 技术架构选型

### 3.1 技术栈总览

| 技术领域       | 选型方案          | 版本要求  | 说明                                     |
| :------------- | :---------------- | :-------- | :--------------------------------------- |
| 编程语言       | JavaScript        | ES6+      | 主要编程语言，排他性使用，不使用 TypeScript |
| 前端框架       | Vue3              | ^3.4.0    | 组合式 API（Composition API）为核心       |
| UI 组件库      | Element-Plus      | ^2.5.0    | 提供表单、表格、弹窗、导航等企业级组件    |
| 构建工具       | Vite              | ^5.0.0    | 极速开发服务器与生产构建                  |
| 路由管理       | Vue Router        | ^4.3.0    | 单页面应用路由控制                       |
| 状态管理       | Pinia             | ^2.1.0    | Vue3 官方推荐状态管理库                   |
| 图标库         | @element-plus/icons-vue | ^2.3.0 | 配套 Element-Plus 的图标组件              |
| HTTP 客户端    | Axios（可选）     | ^1.7.0    | 模拟接口调用（本项目实际使用 Mock 数据）  |
| 数据持久化     | localStorage      | 浏览器原生 | 存储用户操作数据，模拟后端持久化          |
| CSS 预处理     | 原生 CSS / Scoped | —         | 配合 Element-Plus 默认样式                |

### 3.2 技术约束

1. **不得引入任何后端服务**：包括但不限于 Node.js 服务端框架（Express、Koa）、Java 后端、Python 后端、数据库（MySQL、MongoDB 等）。
2. **不得引入 TypeScript**：严格使用 JavaScript 编写全部代码。
3. **数据仅存在于浏览器端**：使用 localStorage 作为持久化方案，所有业务数据在浏览器端存储和操作。
4. **无需配置真实 API 端点**：所有接口请求均以 Mock 方式拦截，不发起实际网络请求。

## 4. 功能需求

本项目分为 **前端用户端** 和 **后台管理端** 两大子系统，共 7 个核心功能模块。

### 4.1 总体功能结构图

```
图书管理系统
├── 前端用户端
│   ├── 登录页面（FR-01）
│   ├── 图书查询页面（FR-02）
│   └── 当前借阅信息页面（FR-03）
└── 后台管理端
    ├── 后台登录页（FR-04）
    ├── 用户信息管理页（FR-05）
    ├── 借阅历史管理页（FR-06）
    ├── 图书信息管理页（FR-07）
    ├── 采购管理页（FR-08）
    └── 系统管理页（FR-09）
```

### 4.2 模块详细功能需求

---

#### 模块一：前端登录页（FR-01）

**功能编码**：FR-01  
**所属子系统**：前端用户端  
**页面路由**：`/login`

**功能描述**：

提供前端用户的登录认证入口，实现身份验证逻辑。

**详细需求**：

| 编号    | 需求项       | 描述                                                         |
| :------ | :----------- | :----------------------------------------------------------- |
| FR-01-01 | 账号输入     | 提供文本输入框，用于输入读者账号（用户名）                   |
| FR-01-02 | 密码输入     | 提供密码输入框，输入内容以掩码（•）形式显示                  |
| FR-01-03 | 验证码输入   | 提供验证码输入框及验证码图片/文本展示区域，校验用户操作合法性 |
| FR-01-04 | 登录按钮     | 点击后校验输入完整性，验证账号密码，成功则跳转至 `图书查询页`，失败则提示错误信息 |
| FR-01-05 | 注册跳转     | 提供"没有账号？立即注册"文字链接，点击跳转至注册页面（本项目可在登录页内实现简化注册弹窗） |
| FR-01-06 | 表单验证     | 账号和密码不得为空；验证码必须与显示一致；提供对应的错误提示 |

**交互规则**：
- 账号密码校验通过 → 存储用户登录态至 localStorage → 跳转 `/book-query`
- 校验失败 → 显示错误提示（Element-Plus Message），清空验证码输入框并刷新验证码
- 点击注册文字 → 弹出注册对话框（El-Dialog），包含用户名、密码、确认密码、电话字段

#### 模块二：图书查询页（FR-02）

**功能编码**：FR-02  
**所属子系统**：前端用户端  
**页面路由**：`/book-query`

**功能描述**：

面向读者的图书查询检索页面，支持条件筛选浏览图书，并可执行借阅操作。

**详细需求**：

| 编号    | 需求项       | 描述                                                         |
| :------ | :----------- | :----------------------------------------------------------- |
| FR-02-01 | 搜索/筛选区  | 提供关键词输入框（支持书名、作者模糊搜索）、图书类型下拉选择、出版社筛选 |
| FR-02-02 | 图书列表表格 | 展示列：图书号、图书类型、图书名称、作者名称、出版社、总数量。每行末尾配置"借阅"操作按钮 |
| FR-02-03 | 借阅操作     | 点击"借阅"按钮，若当前图书总数量 > 0，则生成一条借阅记录，否则提示"该书已全部借出" |
| FR-02-04 | 热门推荐     | 侧边栏展示热门图书排行列表（按借阅次数降序展示前 5 本）      |
| FR-02-05 | 最佳读者     | 侧边栏展示借阅次数最多的前 5 名读者，含用户名和借阅次数      |
| FR-02-06 | 问题反馈     | 侧边栏提供简单的反馈表单（文本域 + 提交按钮），数据保存至 localStorage |
| FR-02-07 | 分页功能     | 图书列表支持分页，默认每页展示 10 条，支持切换页码和每页条数 |

**交互规则**：
- 进入页面时默认加载全部图书列表
- 搜索/筛选条件变更后实时更新列表结果
- 借阅操作成功后刷新图书总数量和借阅历史数据
- 热门推荐和最佳读者数据基于实际借阅记录统计

## 7. 数据结构定义

### 7.1 图书对象（Book）

```javascript
/**
 * @typedef {Object} Book
 * @property {string}  id          - 图书唯一编号（UUID）
 * @property {string}  bookNo      - 图书号（如：BK-20260001）
 * @property {string}  name        - 图书名称
 * @property {string}  category    - 图书分类（如：文学、科技、教育、历史）
 * @property {string}  author      - 作者名称
 * @property {string}  publisher   - 出版社
 * @property {string}  isbn        - ISBN 编号
 * @property {number}  price       - 单价（元）
 * @property {number}  totalCount  - 总数量（馆藏总数）
 * @property {number}  borrowCount - 当前借出数量
 * @property {string}  status      - 图书状态（'available' | 'unavailable'）
 * @property {string}  description - 图书描述
 * @property {string}  createdAt   - 创建时间（ISO 8601 字符串）
 */
```

**示例数据**：

```javascript
{
  id: "b001",
  bookNo: "BK-20260001",
  name: "JavaScript 高级程序设计",
  category: "科技",
  author: "Matt Frisbie",
  publisher: "人民邮电出版社",
  isbn: "978-7-115-54586-0",
  price: 129.00,
  totalCount: 10,
  borrowCount: 2,
  status: "available",
  description: "JavaScript 红宝书第4版",
  createdAt: "2026-01-15T08:00:00.000Z"
}
```

**计算属性**：
- `availableCount`（在馆数量）= `totalCount - borrowCount`
- `status` 当 `totalCount - borrowCount > 0` 时为 `'available'`，否则为 `'unavailable'`

## 8. 交互流程说明

### 8.1 用户登录流程

```
[访问系统] → 判断是否已登录
                ├── 已登录 → 跳转对应首页（用户端 → /book-query；管理端 → /admin/user-manage）
                └── 未登录 → 显示登录页
                              → 用户输入账号、密码、验证码
                              → 点击"登录"
                              → 前端校验输入完整性
                              → 从 localStorage 读取用户列表并匹配
                              → 匹配成功 → 存储登录态 → 跳转
                              → 匹配失败 → 提示错误信息 → 刷新验证码
```

### 8.2 图书借阅流程

```
[图书查询页] → 浏览/搜索图书列表
              → 点击某本图书的"借阅"按钮
              → 前端校验：
                  ├── 该图书记录是否存在 → 否 → 提示错误
                  ├── availableCount > 0 → 否 → 提示"该书已全部借出"
                  └── 用户是否有超期未还图书 → 是 → 提示"请先归还超期图书"
              → 创建借阅记录（borrowDate = 今天, dueDate = 今天 + config.borrowDays）
              → 更新图书 borrowCount +1
              → 更新用户 borrowCount +1
              → 全部写入 localStorage
              → 提示"借阅成功" → 刷新页面数据
```

### 8.3 还书流程

```
[当前借阅信息页] → 浏览当前用户的借阅记录
                → 点击某条记录的"还书"按钮
                → 弹出确认对话框："确认归还《XXX》吗？"
                → 用户点击"确认"
                → 更新借阅记录：returnDate = 今天, status = 'returned'
                → 更新图书 borrowCount -1
                → 全部写入 localStorage
                → 提示"归还成功" → 刷新页面数据
```

### 8.4 图书新增/编辑流程

```
[图书信息管理页] → 点击"新增图书"按钮
                  → 弹出表单对话框（El-Dialog）
                  → 填写：书名、分类(下拉选择)、作者、出版社、ISBN、价格、总数量、描述
                  → 点击"保存"
                  → 前端校验：
                      ├── 必填项是否已填写 → 否 → 显示错误提示
                      ├── ISBN 是否重复 → 是 → 提示"ISBN 已存在"
                      └── 价格/数量是否为正数 → 否 → 提示错误
                  → 生成新 Book 对象，id = UUID
                  → 写入 localStorage
                  → 记录操作日志
                  → 提示"新增成功" → 关闭弹窗 → 刷新列表

[编辑流程] → 点击某行"编辑"按钮
            → 弹出预填充数据的表单对话框
            → 修改内容 → 保存 → 校验同新增 → 更新 localStorage → 记录日志 → 刷新列表
```

### 8.5 删除确认流程

```
[任意管理页面] → 点击某行"删除"按钮
                → 弹出 ElMessageBox.confirm("确认删除该记录吗？此操作不可恢复。")
                → 用户点击"取消" → 关闭弹窗，无操作
                → 用户点击"确认"
                → 前置校验（如图书删除前检查是否有未归还借阅记录）
                → 从 localStorage 中移除对应数据
                → 记录操作日志
                → 提示"删除成功" → 刷新列表
```

### 8.6 路由守卫流程

```
[用户访问任意 URL] → router.beforeEach 拦截
                    → 判断目标路由是否需要登录（meta.requiresAuth）
                        ├── 不需要 → 直接放行
                        └── 需要
                            → 检查 localStorage 中是否有 bms_currentUser

                                        ├── 不匹配 → 重定向至对应首页
                                        └── 匹配 → 放行
```

### 9.2 路由守卫逻辑

```javascript
// 伪代码示例
router.beforeEach((to, from, next) => {
  const currentUser = JSON.parse(localStorage.getItem('bms_currentUser'));
  const isAuthenticated = !!currentUser;
  const requiredRole = to.meta.role;

  if (to.meta.guest) {
    // 游客页面（登录页），已登录则跳转对应首页
    if (isAuthenticated) {
      if (currentUser.role === 'admin') next('/admin/user-manage');
      else next('/book-query');
    } else {
      next();
    }
  } else if (to.meta.requiresAuth) {
    // 需要登录的页面
    if (!isAuthenticated) {
      // 未登录，跳转对应登录页
      if (to.path.startsWith('/admin')) next('/admin/login');
      else next('/login');
    } else if (requiredRole && currentUser.role !== requiredRole) {
      // 角色不匹配
      if (currentUser.role === 'admin') next('/admin/user-manage');
      else next('/book-query');
    } else {
      next();
    }
  } else {
    next();
  }
});
```

1. 组件通过 Pinia Store 的 actions 发起数据变更

3. Store 初始化时从 localStorage 读取数据加载至内存
4. 若 localStorage 无数据（首次访问），使用预置 Mock 数据初始化

系统首次加载时，检测 localStorage 是否存在 `bms_books` 等关键 key：
- **已存在**：直接加载现有数据
- **不存在**：从 `src/mock/` 目录下的数据文件中读取预置 Mock 数据，写入 localStorage

预置 Mock 数据规模：
- 图书：≥ 15 条
- 用户：≥ 10 条（含 1 个管理员账号）
- 借阅记录：≥ 8 条
- 采购记录：≥ 5 条

### 13.2 UUID 生成方案

```javascript
// 使用 crypto.randomUUID() 或 polyfill 生成唯一ID
function generateId() {
  if (crypto.randomUUID) {
    return crypto.randomUUID();
  }
  // fallback: 时间戳 + 随机数
  return Date.now().toString(36) + Math.random().toString(36).substr(2);
}
```

### 13.3 Base64 编解码方案

```javascript
// 密码存储使用 btoa / atob
function encodePassword(password) {
  return btoa(unescape(encodeURIComponent(password)));
}

function decodePassword(encoded) {
  return decodeURIComponent(escape(atob(encoded)));
}
```
